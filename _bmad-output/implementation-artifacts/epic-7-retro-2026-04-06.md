## Struggles & Lessons Learned

### Timer cleanup missed again — same pattern as Epic 5

- **7-5**: `copyTimer` in `GameHeader.svelte` not cleared on component teardown (review patch).
- **7-6**: `undoTimer` clear-on-End-Round was missing (review patch); `undoTimer` not cleared on session end path (review patch).
- Epic 5 retro called this out explicitly: "Every `setTimeout`/`setInterval` in a Svelte component needs a paired `onDestroy` cancellation — treat it as a checklist item before marking a story ready for review."
- **Status:** The lesson is documented but not being applied. It has now appeared in five stories across two epics.
- **Next action:** Add "onDestroy clears all timers" as a first-pass checklist item before any story goes to review.

### Double-invocation bugs on session lifecycle paths (7-6)

Two separate bugs in 7-6 allowed the same lifecycle callback to fire twice:
1. `onSessionEnded` could be triggered by both the WS `session:end` message AND the DELETE fetch response completing — both paths wired to the same handler without a guard.
2. `handleEndRound` starting a second `undoTimer` without first clearing the previous one if the host tapped End Round twice.

Both required guards (`sessionEnded` flag; `clearTimeout(undoTimer)` before starting a new timer). The pattern is the same as the Epic 5 concurrent-call bugs (5-5 `/round/claim` race, 5-6 `reinitSdk` re-entry).

**Lesson:** Any function that kicks off an async sequence or sets a timer must check whether one is already in-flight before proceeding. Write the guard before the logic.

### Z-index collision: toast obscured by overlay backdrop

When a user tapped gear during the 2s undo window, the HostControlsOverlay backdrop (z-index 149) rendered over the toast (which was at a lower index). Fix was to raise the toast to z-index 200. The collision was predictable from the z-index table in the story spec but wasn't caught during initial implementation.

**Lesson:** When a story spec lists a z-index table, that table is a collision checklist — verify each new element's position in the stack during implementation, not at review.

### CSS pixel mismatches caught in review (7-4)

Host-pill font-size implemented as `0.4375rem`; spec required `0.6875rem`. These are not approximations — the story spec was explicit. The miss cost a review round.

**Lesson:** When a spec gives exact pixel/rem values, those values are copy targets, not guidelines.

### Error swallowing deferred for the third consecutive epic

`handlePlayPause` and `handleNext` in HostRoomPage swallow fetch errors silently. This was flagged in Epic 5 (in HostControlsPanel), carried into Epic 6 preparation notes, and is now deferred again from 7-6. It will cause invisible failures when the host loses connectivity mid-game.

**Status:** Persistent debt. Must address in Epic 6 hardening — it is a production observability issue, not a polish item.
