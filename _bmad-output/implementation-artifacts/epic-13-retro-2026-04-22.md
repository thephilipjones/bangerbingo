## Successes

1. **Systematic debt clearance achieved** — Epic 13 closed deferred items dating back to Epics 1 (1-1, 1-2), 3 (3-1, 3-2, 3-5), 4 (4-3), 8 (8-3, 8-5), 9 (9-1, 9-3), and 12 (12-2). The primary goal of the epic was met completely.

2. **Pre-existing implementations surfaced** — Three "deferred" items were discovered already resolved during 13-3, 13-4, and 13-5: `roomSockets` eviction, `startRefreshScheduler` test guard, and both cookie signing and `playlistId` sanitization. Discipline in reading current code before speccing them out saved unnecessary work.

3. **Real staging bug fixed with root-cause depth** — 13-8's trigger was a verified staging incident (session JKAG, duplicate tiles in a winning line). Investigation identified two adjacent design flaws simultaneously: all players sharing the same 25 songs (not true bingo behavior) and the downrank vs. exclude played-songs model. Net code decrease, simpler model, correct behavior.

4. **Security review caught an unlisted injection point** — The 13-5 code review identified an identical `playlistId` path traversal vulnerability in `POST /api/rooms/:code/round` that wasn't in the original ACs. Patched in the same story. This is the review layer doing its job.

5. **Win jingle shipped** — First audio feature in the app. Web Audio API approach: no files, no CDN, triggered by a user-gesture chain so autoplay policy isn't an obstacle. 8 unit tests, three presets.

6. **13-7 identity flow is a genuine UX improvement** — `/host` URL for dashboard, "Join a Session" button, OAuth callback redirects to `/host`, 4003 fallback from lobby to guest join. Well-scoped and cleanly executed.

7. **Test quality improved meaningfully** — Seeded `Math.random` in the `generateCards` test (LCG stub), structural assertions replacing wall-clock timing in the host-disconnect test.

## Challenges

1. **Stale deferred-work.md** — At least 3 items in deferred-work.md were discovered already resolved. The list is written atomically at review time but never pruned as the codebase evolves. This pattern wastes story-spec and verification effort in every debt epic.

2. **13-8 scope expanded significantly** — The story began as a single-line dedup fix. Correct engineering judgment expanded it to 6 changes and 10 ACs. This is good — the resulting model is cleaner — but it exceeded the "10 ACs / 4+ concerns" split threshold the team committed to in Epic 12's retrospective. The expansion was intentional and documented, but it wasn't flagged at spec time.

3. **12-2 on-device journeys still unverified** — The HIGH priority action item from Epic 12 retro (run iPhone + Bluetooth + Connect speaker journeys A–G) carried through Epic 13 without completion. No story or commit addresses it.

4. **Double-POST on mobile resume** — Epic 12 retro explicitly listed this as "Owner: 13-3 micro-fixes bundle." It was not included in the 13-3 spec and remains open in `deferred-work.md`.

5. **`square:auto-marked` test flake confirmed but unresolved** — Surfaced in 13-4 completion notes and confirmed again in 13-5 test runs (~2/4 full-suite runs fail under parallelism). The flake exists, is reproducible, and has been diagnosed as a vitest cross-file timing issue. It now has two confirmed sightings with no fix.

6. **Test counts non-monotonic mid-epic** — 507 → 509 → 522 → 517 → 517 → 519 → (8 added) → 529. The dip from 522 to 517 suggests 13-2 and 13-3 were worked in different orders or against different branch states. Not a correctness concern, but the variance makes it harder to reason about test delta per story.

## Key Insights

1. **Deferred-work.md drifts silently** — Items get added at review time and removed only when a story explicitly closes them. But some fixes land as patches in unrelated stories (e.g., Story 5-7 closed the `startRefreshScheduler` guard years before 13-4 tried to address it). An annual or per-epic "deferred list audit" pass would prevent stale entries from accumulating spec-effort.

2. **Debt epics surface unexpected scope** — 13-8 shows the pattern: a 1-line bug → 2-day redesign, producing net code decrease and correct behavior. When investigating root causes in a debt epic, budget for expansion. The right call is nearly always to fix completely rather than narrowly.

3. **Security code review catches cross-cutting issues that ACs miss** — The 13-5 story ACs targeted three specific endpoints. The review caught a fourth. This is structural: ACs are written from a list of known-deferred items; code review scans the surrounding surface. Both layers are needed.

4. **Web Audio API is a valid audio primitive for bingo** — No files, no loading, no CDN, triggered by gesture chain, teardown is automatic when nodes finish. If more game audio is ever needed (tile mark, round start, countdown), the `winAudio.ts` pattern generalizes.

5. **`/host` as the dedicated host URL is the right direction** — The previous behavior (`/` polymorphic on auth state) caused real confusion for multi-device testing. The split to `/host` (authenticated) and `/` (join form always) is cleaner and pairs well with the 4003 fallback from LobbyPage.
