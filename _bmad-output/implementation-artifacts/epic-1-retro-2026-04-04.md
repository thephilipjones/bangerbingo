## Epic Summary

Established the full project skeleton and delivered working Spotify PKCE auth with session persistence and a proactive token refresh scheduler with degraded-mode signalling. The greenfield start required establishing conventions (TypeScript config, test patterns, project structure) that every subsequent epic builds on.

---

## What Went Well

- **PKCE implementation was clean on first pass.** RFC 7636 requirements (code_verifier, code_challenge, S256) were well-understood from the spec and executed correctly without back-and-forth.
- **EventEmitter decoupling (refresh → WS).** The `authEvents` EventEmitter in `refresh.ts` keeps the dependency direction clean — refresh module does not know about WebSockets. This was a good architectural call that paid off immediately in Epic 3.
- **Test isolation pattern landed well.** `initDb(':memory:')` in `beforeEach` became the canonical pattern adopted by all subsequent stories without variation.
- **Crash-fast env validation.** The `config.ts` startup validation saved debugging time in later stories — any missing var surfaces immediately with a clear message.

## Struggles & Lessons Learned

### TypeScript `moduleResolution` mismatch
- **What happened:** Initial `tsconfig.json` used `moduleResolution: node`, which doesn't allow `.ts` extensions in imports (required by `tsx`). Fixed by switching to `moduleResolution: bundler` with `allowImportingTsExtensions: true` and `noEmit: true`.
- **Lesson:** Establish the TypeScript config baseline carefully at story 1.1 and document it. Later stories inherited this without issue, but a wrong baseline here would have compounded across the whole project.

### Server started unconditionally in test environment
- **What happened (1.2):** Two test files both imported `index.ts`, which called `serve()` at module load. EADDRINUSE errors in tests until `serve()` was guarded with `if (config.nodeEnv !== 'test')`.
- **Lesson:** Any server/daemon started at module load must be environment-gated. Apply this pattern proactively in all future stories that import `index.ts` in tests.

### `vi.runAllTimersAsync()` caused infinite setInterval loops
- **What happened (1.2):** Scheduler tests using `vi.runAllTimersAsync()` re-triggered the `setInterval` endlessly. Fixed by using `clearInterval(interval)` before assertions and `vi.advanceTimersByTimeAsync(60_000)` for a single tick.
- **Lesson:** For recurring timers in tests, advance time explicitly and then clear — never call `runAllTimers` on an interval.

## Review Feedback Patterns

Across both stories, **4 patches were applied** and **~15 items deferred**. Common patch themes:

1. **Missing error handling in async onMount handlers** — non-401 errors leave UI stuck at 'loading' (1.1). Pattern: always wrap async calls in onMount with try/catch or `.catch()`.
2. **Global state mutated in tests without cleanup** — `global.fetch = vi.fn()` without `afterEach` restore (1.1). Pattern: always restore mocks in `afterEach`; use `vi.restoreAllMocks()`.
3. **Missing input validation before trusting external JSON** — `tokenRes.json()` can throw on malformed Spotify responses; `updateHostTokens` silently no-ops when userId missing (1.1, 1.2). Pattern: validate shape of external API responses before destructuring.
4. **Scheduler async callback swallows errors silently** — rejected promise from `getAllHosts()` lost with no logging (1.2). Pattern: always add a `.catch(console.error)` or try/catch to async setInterval callbacks.

## Technical Debt Carried Into Epic 2+

High-priority for future hardening (in priority order):
- `startRefreshScheduler()` not guarded by `nodeEnv !== 'test'` — leaks a live interval across test files
- Concurrent scheduler ticks can race on the same host token — needs an in-progress Set guard before Epic 5 adds more load
- `SESSION_SECRET` validated at startup but never used to sign cookies — wire actual signing before any non-friends deployment

Lower priority (fine for personal/friends MVP):
- Tokens stored as plaintext SQLite — acceptable at this scale
- No `AbortController` timeout on Spotify fetch calls
- No jitter in backoff, no 4xx vs 5xx retry discrimination
